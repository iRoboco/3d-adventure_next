### 🔍 Подробное разъяснение математики и геометрии процесса

#### 1. ♦️ Система координат и грани куба
В Axmol используется **правосторонняя система координат** (как в OpenGL):
- `+X` → вправо
- `+Y` → вверх
- `+Z` → "к зрителю" (вперёд)

Каждый воксель представляет собой единичный куб, чьи вершины лежат в диапазонах:
`[x, x+1]` по оси X, `[y, y+1]` по Y, `[z, z+1]` по Z.
При генерации мы оперируем **локальными координатами чанка**, а позиция самого чанка задаётся позже через `Node::setPosition3D(chunkToWorld(key))`.

#### 2. ♦️ Face Culling (Отсечение невидимых граней)
Куб имеет 6 граней. В сплошном массиве блоков **внутренние грани никогда не видны**. Рендерить их бессмысленно:
- Без отсечения: `16×256×16 × 6 граней = 393 216` квадов на чанк.
- С отсечением: в среднем ~`15 000–25 000` (экономия >90% вершин и индексов).

Логика `isSolid` проверяет 6 соседей. Если сосед `!= BLOCK_AIR`, грань между ними **не генерируется**. Проверка соседей за границей чанка (`neighborX0`, `neighborZ1` и т.д.) гарантирует, что на стыке двух чанков не появится "двойная стена" или щель.

#### 3. ♦️ CCW-порядок вершин (Counter-Clockwise)
Графический конвейер GPU определяет, куда "смотрит" грань, по порядку обхода вершин:
- `v0 → v1 → v2` → векторное произведение `(v1-v0) × (v2-v0)` даёт **нормаль**.
- Если вершины идут **против часовой стрелки** (CCW) относительно наблюдателя, грань считается лицевой.
- Обратные грани автоматически отсекаются настройкой `glEnable(GL_CULL_FACE)` (включена по умолчанию в Axmol).
- Именно поэтому в `addFace` порядок `{v0, v1, v2, v3}` строго фиксирован для каждой из 6 сторон. Нарушение порядка приведёт к "дырам" или чёрным граням при включённом освещении.

#### 4. ♦️ UV-развёртка текстурного атласа
Одна текстура `terrain_atlas.png` содержит сетку тайлов (например, `16×16`).
- Вся текстура нормирована: `U ∈ [0,1]`, `V ∈ [0,1]`.
- `blockId` определяет индекс тайла. Математика:
  ```
  col = blockId % 16
  row = blockId / 16
  u_min = col / 16.0
  v_min = row / 16.0
  tile_size = 1.0 / 16.0
  u_max = u_min + tile_size
  v_max = v_min + tile_size
  ```
- UV координаты привязываются к вершинам квада так, чтобы текстура "натягивалась" ровно на грань блока без искажений. `uv[0], uv[1]` → нижний-левый угол тайла, `uv[2], uv[3]` → верхний-правый.

#### 5. ♦️ Триангуляция и индексный буфер
GPU рендерит **только треугольники**. Квадратная грань разбивается на два треугольника:
```
v0 ───── v1
 │      /│
 │    /  │
 │  /    │
v3──────v2
```
- Треугольник 1: `v0 → v1 → v2` (индексы `baseIdx+0, +1, +2`)
- Треугольник 2: `v0 → v2 → v3` (индексы `baseIdx+0, +2, +3`)
Диагональ `v0→v2` общая. Индексный буфер (`ax::IndexArray`) позволяет переиспользовать вершины: вместо хранения 6 вершин на грань мы храним 4, а треугольники ссылаются на них по индексам. Это экономит ~33% памяти видеокарты и ускоряет кэш-локальность.

#### 6. ♦️ Сборка `ax::Mesh`
Axmol требует раздельные массивы атрибутов (`positions`, `normals`, `texcoords`), хотя в `ChunkVertex` они интерливингованы (чередуются). Функция `createMesh` выполняет **деинтерливинг** перед вызовом `ax::Mesh::create()`. Движок автоматически:
1. Создаёт `VBO` (Vertex Buffer Object) и `IBO` (Index Buffer Object)
2. Рассчитывает `AABB` (ограничивающий параллелепипед) для Frustum Culling
3. Копирует данные в VRAM и готовит команду отрисовки (`MeshCommand`)
